# 差异化

open-design 能做到、而 Claude Design（以及其他 AI 设计工具）不能或不会做的事情。

## 结构性优势（由我们的架构天然带来）

| #   | 优势                   | 为什么竞争对手很难匹配                                     |
| --- | ---------------------- | ---------------------------------------------------------- |
| 1   | 零供应商锁定           | 他们只提供 Opus；我们通过 pi-ai 运行任意模型（BYOK）       |
| 2   | 设计永不离开笔记本电脑 | 对法律/医疗/金融场景的合规友好                             |
| 3   | 无上限、无订阅         | 本地模型 = 实际上免费；云模型 = 只按 token 付费            |
| 4   | 可 fork、可 hack       | 自定义 system prompts、自定义 exporters — 闭源工具不会允许 |
| 5   | 无 Canva 依赖          | 直接 PPTX/HTML/PDF；没有第二份订阅，也没有第二次数据泄露   |

## 为 v0.2 — v0.5 选定的杀手级功能

按 `(impact × ease)` 评分。这些会和八个 Claude Design 演示一起交付。

| 功能                                                                        | 阶段 | 为什么它会传播                                       |
| --------------------------------------------------------------------------- | ---- | ---------------------------------------------------- |
| **三栏模型 A/B 竞速**（同一提示词 → Opus vs GPT-5 vs Gemini 并排）          | v0.2 | 天然适合发推截图；“哪个模型设计最好”的争论会自我推动 |
| **CLI 模式**（`open-design "make me a landing page" -m gpt-5 -o out.html`） | v0.3 | HN 首页诱饵；可集成进 shell、Makefiles、CI           |
| **Steal URL Style**（粘贴任意 URL → 学习美学 → 应用到你的内容）             | v0.3 | 30 秒演示视频格式；立即有实用价值                    |
| **Reverse Redesign**（指向任意丑网站 → AI 重设计）                          | v0.4 | 经典 before/after Twitter 形式                       |
| **本地 Ollama 零成本预设**                                                  | v0.4 | “永远免费，无需 API key”的标语                       |
| **多 IDE 交付**（Cursor / Cline / Aider / Claude Code 等）                  | v0.5 | 连接整个 AI 编程生态                                 |

## 暂缓路线图（1.0 之后）

| 功能                                    | 暂缓原因                        |
| --------------------------------------- | ------------------------------- |
| Skills marketplace（社区 prompt packs） | 先需要用户基础                  |
| GitHub PR 设计预览 bot                  | 需要后端服务 — 违反“无后端”原则 |
| Figma import                            | 工程投入与回报不划算            |
| 实时多人编辑                            | 违反本地优先架构                |
| Git-aware design diff                   | 有用但不具传播性                |
| Codebase live sync（file watcher）      | 打磨项，不是差异化              |
| MCP server interface                    | 技术干净，但受众较小            |

## 品牌层定位（候选）

- “没有锁定的 Claude Design”
- “你的设计、你的模型、你的机器。”
- “开源 AI 设计 — 本地、多模型、永远属于你。”

## 反营销主张

即使为真，我们也不会这样说：

- ❌ “比 Claude Design 更好” — 我们不是更好，我们是不同
- ❌ “比 Figma 更快” — 产品维度正交
- ❌ “替代你的设计师” — 我们是增强，而不是替代
